# CF-Workers-SUB-Next 项目规划

> 新一代 Cloudflare Workers 订阅汇聚与规则生成工具

---

## 一、现状分析

### 现有项目（CF-Workers-SUB）

| 维度 | 现状 |
|------|------|
| 部署方式 | Cloudflare Workers / Pages + KV |
| 认证方式 | Token 访问（无 UI 界面） |
| 订阅管理 | 通过编辑环境变量或 KV 编辑页面添加链接 |
| 规则格式 | `.ini` 外部配置（subconverter 格式） |
| 转换后端 | 依赖第三方 SUBAPI（sub.cmliussss.net） |
| 规则来源 | ACL4SSR 的 `.list` / `.ini` 规则 |
| 作者维护 | 上游已停更 |

### 核心痛点

1. **.ini 规则格式已过时** — Clash Meta / Mihomo 原生支持的是 `rule-providers`（YAML 格式），.ini 是 subconverter 时代的产物，需要额外转换层
2. **无管理界面** — 每次修改订阅链接需要进 KV 或环境变量，操作门槛高
3. **依赖第三方转换后端** — 节点信息经第三方服务器，存在隐私风险
4. **规则不可选** — 固定一套 ACL4SSR 多国规则，无法按需切换

## 二、现代规则格式对比

### 旧格式（.ini - subconverter 外部配置）

```ini
[custom]
ruleset=🚀 节点选择,https://.../ProxyLite.list
ruleset=🎯 全球直连,https://.../LocalAreaNetwork.list
ruleset=🛑 全球拦截,https://.../BanAD.list
```

- 需要 subconverter 后端解析
- 规则文件是 `.list` 纯文本格式
- 无法直接用于 Clash Meta / Mihomo

### 新格式（YAML - rule-provider）

```yaml
# rule-provider 定义
rule-providers:
  ProxyLite:
    type: http
    behavior: classical
    url: "https://raw.githubusercontent.com/ACL4SSR/ACL4SSR/master/Clash/Providers/ProxyLite.yaml"
    interval: 86400
    path: ./rules/ProxyLite.yaml

# 规则引用
rules:
  - RULE-SET,ProxyLite,🚀 节点选择
  - RULE-SET,BanAD,🛑 全球拦截
  - GEOIP,CN,🎯 全球直连
  - MATCH,🐟 漏网之鱼
```

- Clash Meta / Mihomo / OpenClash 原生支持
- 支持自动更新规则（interval 参数）
- YAML 格式更规范，可读性更强
- ACL4SSR 已经提供了 YAML 版本（`/Clash/Providers/` 目录）

## 四、功能清单

## 五、预置规则模式

| 模式名称 | 说明 | 规则集 |
|---------|------|--------|
| **🌐 全球直连** | 国内直连 + 国外代理 | ProxyLite + ChinaDomain + ChinaCompanyIp + GEOIP_CN |
| **🎬 媒体解锁** | 针对流媒体优化 | 以上 + Netflix + YouTube + Disney + 国外媒体 |
| **🤖 AI 加速** | 针对 AI 服务优化 | 以上 + OpenAI + Claude + Copilot + GitHub |
| **🎮 游戏加速** | 低延迟游戏节点 | 以上 + SteamCN + 游戏服务器 |
| **📱 全功能** | 默认全部规则 | 以上所有 + 广告拦截 + 应用净化 |
| **✏️ 自定义** | 用户自行勾选 | 自由组合所有规则集 |

每个模式对应不同的：
- `rule-providers` 列表（引用哪些规则集）
- `rules` 顺序（优先级排序）
- `proxy-groups` 策略组（节点分组方式）

## 八、Cloudflare Workers 技术可行性

| 要求 | 可行性 | 说明 |
|------|--------|------|
| 密码登录 | ✅ 简单 | Workers 支持 Session/Cookie，密码用环境变量配置 |
| Web UI | ✅ 可行 | 单页 HTML 内嵌在 Worker 返回中，无需额外部署 |
| KV 存储 | ✅ 可行 | 免费 1GB 存储，1000 次写入/天，足够个人使用 |
| 订阅抓取 | ✅ 可行 | Workers 支持 fetch API 抓取外部订阅 |
| YAML 生成 | ✅ 可行 | 纯字符串拼接，Workers 原生支持 |
| 规则引用 | ✅ 可行 | 直接输出 rule-provider 的 HTTP URL 引用 |
| 节点解析 | ⚠️ 部分可行 | 需要解析 vmess/vless/trojan/ss/hysteria2 等链接 |

**关键限制：**
- Workers 免费计划每天 10 万次请求，CPU 时间 10ms/请求
- 节点解析需要自己实现协议解析器（vmess/vless/trojan/ss/hysteria2）
- 无法直接测速（TCP 连接限制）
- KV 写入有配额限制（每天 1000 次）

## 十一、总结与建议

**结论：项目可行。**

核心优势：
1. 去掉对第三方 subconverter 的依赖，节点信息不再经他人服务器
2. 使用 YAML rule-provider 原生格式，OpenClash 直接支持，无需额外转换
3. 密码保护的 Web UI，管理订阅和切换规则模式都很方便
4. 多规则模式可切换，适配不同场景

**建议先行：** 先做 MVP（密码登录 + 订阅管理 + 基础 YAML 生成 + 2-3 种规则模式），验证可用后再逐步加高级功能。
